Game Flow

How a flip works

Four on-chain steps, powered by BLS randomness.

🎲

Player creates game

Calls start_game(guess, stake). Guess is 0 (heads) or 1 (tails). Stake must be within min_stake and max_stake.

🔏

House signs the game ID

House signs game_id using its BLS12-381 private key. The signature is the source of randomness — it cannot be predicted before the game exists.

⚖️

On-chain settlement

finish_game(game_id, bls_sig) verifies the BLS signature via bls12381_min_pk_verify. Blake2b256 hash → even byte = heads, odd = tails.

💰

Payout

Winner receives 2× stake minus fee (taken in basis points from base_fee_in_bp). Fees accumulate in house balance, withdrawable by admin via HouseCap.

Verifiable

BLS signature verified fully on-chain via bls12381::bls12381_min_pk_verify. Anyone can re-derive the result.

Unpredictable

House can't pre-compute signatures for future game IDs. The game object doesn't exist until the player creates it.

No oracle needed

BLS12-381 is a native Sui framework primitive. Zero external dependencies for randomness.
